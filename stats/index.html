Data Project Fundamentals

6 Questions to Ask About Any New Dataset
  1. How many observations (rows) and variables (columns) are there?
  2. What are the data types of each column (numeric, character, factor)?
  3. Are there missing values — and if so, what do they mean?
  4. What does each row represent? (the entity / unit of analysis)
  5. Where did the data come from and can you trust it?
  6. What time period does it cover?
The Process
  • Data represents real-world phenomena — always ask "what does a row mean?"
  • 80% of the work is in preparation and tidying
  • Only 20% is analysis and visualization

Section 04

dplyr — Data Manipulation Package

What is dplyr? dplyr is R's equivalent of Python's pandas. It gives you a clean, readable grammar for manipulating data frames. It's part of the tidyverse. Always load it with library(dplyr).
FunctionWhat it doesExample
select()Choose columns to keepselect(df, col1, col2)
filter()Keep rows that match a conditionfilter(df, amount > 100)
mutate()Create a new column (or overwrite one)mutate(df, ratio = a/b)
summarize()Collapse rows into summary statisticssummarize(df, avg = mean(x))
group_by()Group rows before summarizinggroup_by(df, region)
arrange()Sort rowsarrange(df, desc(amount))
The Pipe Operator |>

The pipe passes the left-hand result as the first argument to the right-hand function. It makes multi-step operations read like a sentence.

library(dplyr)

# Without pipe (hard to read)
arrange(summarize(group_by(filter(df, region == "East"), region),
  total = sum(sales)), desc(total))

# With pipe (clean and readable)
df |>
  filter(region == "East") |>
  group_by(region) |>
  summarize(total = sum(sales)) |>
  arrange(desc(total))
select() — Pick Columns
library(readr)
sales <- read_csv("salesdata.csv")

# Keep only two columns
sales |> select(region, amount)

# Drop a column with minus
sales |> select(-id)
filter() — Subset Rows
# Single condition
sales |> filter(amount > 500)

# Multiple conditions (AND)
sales |> filter(amount > 500,
                region == "West")

# OR condition
sales |> filter(region == "East" |
                region == "West")
mutate() — New Columns
# Create a new column
sales |> mutate(tax = amount * 0.08)

# Multiple new columns at once
sales |> mutate(
  tax    = amount * 0.08,
  total  = amount + tax
)
group_by() + summarize()
# Total sales by region
sales |>
  group_by(region) |>
  summarize(
    total = sum(amount),
    avg   = mean(amount),
    n     = n()
  )

Section 06

Data Cleaning & Preparation

80 / 20 Rule 80% of any real data project is cleaning and preparation. Only 20% is analysis. The tools below are what you use to assess, handle, and fix messy data before visualization or modeling.
Identifying Missing Values
# Is each value NA?
is.na(df$column)                    # → vector of TRUE/FALSE

# How many NAs in one column?
sum(is.na(df$column))               # → single integer

# How many NAs in every column at once?
colSums(is.na(df))                  # → named vector per column

# Which rows have NO missing values?
complete.cases(df)                  # → TRUE/FALSE per row
df[complete.cases(df), ]            # → keep only complete rows

# Which rows HAVE missing values?
df[!complete.cases(df), ]           # → isolate the bad rows
5 Missing Value Strategies
StrategyWhen to use itR approach
Manual replacementLogic is obvious (e.g., season from a date)Assign directly: df$col[condition] <- value
InterpolationTime series with forward/backward fillzoo::na.locf() or tidyr::fill()
RemovalOnly a few rows missing & data is not criticalna.omit(df) or filter(complete.cases(df))
ImputationNumeric column; missing at randomdf$col[is.na(df$col)] <- mean(df$col, na.rm=TRUE)
Predefined valueCategory column with a known "unknown" codeAssign a factor level like "Unknown"
Factor Variable Conversion

Categorical data is often stored as integer codes (1, 2, 3, 4). You need to convert these to factors so R treats them as categories, not numbers. This affects charts, models, and summaries. 

library(readr)
bikeshare <- read_csv("bikesharedailydata.csv")

# Convert season: 1=Winter, 2=Spring, 3=Summer, 4=Fall
bikeshare$season <- factor(
  bikeshare$season,
  levels = 1:4,
  labels = c("Winter", "Spring", "Summer", "Fall")
)

# Convert year: 0="2011", 1="2012"
bikeshare$yr <- factor(
  bikeshare$yr,
  levels = c(0, 1),
  labels = c("2011", "2012")
)
Why this matters Without factor conversion, R will treat season 4 as "twice as much" as season 2 in a model. With factor(), R knows it's a category, not a number. Always check your data dictionary to understand what each code means.
Manual Season Fix from Date
# If season column has NAs, derive it from the date
bikeshare$season[
  is.na(bikeshare$season) &
  as.numeric(format(bikeshare$dteday, "%m")) %in% c(12, 1, 2)
] <- "Winter"

# Or use dplyr's mutate + case_when
library(dplyr)
bikeshare <- bikeshare |>
  mutate(season = case_when(
    is.na(season) & month %in% c(12,1,2) ~ "Winter",
    is.na(season) & month %in% c(3,4,5)  ~ "Spring",
    is.na(season) & month %in% c(6,7,8)  ~ "Summer",
    is.na(season) & month %in% c(9,10,11) ~ "Fall",
    TRUE ~ season
  ))
Data Integrity: Accurate Counts

When you title a chart "208 countries," you need to know the actual count of non-missing rows — not just nrow().

# Total rows (includes NAs)
nrow(internet_data)          # e.g., 222

# Actual observations (no NAs)
sum(!is.na(internet_data$`2025`))  # e.g., 208

# Missing count
sum(is.na(internet_data$`2025`))   # e.g., 14

# So title should say "(208 countries)" not 222

Section 08

Writing Custom Functions

Why write your own functions? When you need to repeat the same operation many times, wrap it in a function instead of copying and pasting code. Functions also make your code easier to test and reuse.
Function Structure
# Part 1 — define the function
myfunction <- function(myparameter) {
  return(myparameter + 2)
}

# Part 2 — call the function
myfunction(5)    # → [1] 7
myfunction(10)   # → [1] 12
Adding Input Validation

What happens if someone passes a string instead of a number? Add a check with is.numeric():

myfunction <- function(myparameter) {
  if (!is.numeric(myparameter)) {
    stop("Input must be numeric")
  }
  return(myparameter + 2)
}

myfunction("hello")   # → Error: Input must be numeric
myfunction(8)         # → [1] 10
Multiple Arguments
# Function with two parameters
addTogether <- function(x, y) {
  return(x + y)
}

addTogether(3, 4)          # → [1] 7
addTogether(x=3, y=4)      # same — named arguments (alternative)
addTogether(3, "hello")    # → Error (type mismatch)
Built-in Functions You Should Know
FunctionWhat it does
toupper(x)Convert string to uppercase
mean(x, na.rm=TRUE)Average, skip NAs
is.numeric(x)Returns TRUE/FALSE — is x a number?
is.na(x)Returns TRUE/FALSE — is x missing?
sqrt(x)Square root
Pseudocode → Real Code Workflow

Before writing code, sketch the logic in plain English (pseudocode), then translate:

# Pseudocode:
# function "averageTwo" takes two numbers x and y
# adds them together, divides by 2, returns result

averageTwo <- function(x, y) {
  if (!is.numeric(x) | !is.numeric(y)) stop("Both inputs must be numeric")
  return((x + y) / 2)
}

averageTwo(10, 20)   # → [1] 15

sapply — Apply a Function Over a Vector

The apply family Instead of writing a for loop to apply a function to every element, R has the apply family. sapply() applies a function over a vector or list and returns a simplified result (usually a vector).
sapply() Syntax
# Pattern: sapply(vector_or_list, function)

# Define a function
takemean <- function(x) { mean(x, na.rm = TRUE) }

# Apply it over a named list of vectors
my_data <- list(a = c(1,2,3), b = c(10,20), c = c(5,5,5,5))
sapply(my_data, takemean)
## a b c ## 2 15 5

Flexdashboards

What is a Flexdashboard? A Flexdashboard is an R Markdown document that publishes a group of related visualizations as a dashboard web page. It can be static (plain HTML) or dynamic (powered by Shiny).
Setup
  1. Install the package: install.packages("flexdashboard")
  2. In RStudio: File → New File → R Markdown → From Template → Flex Dashboard
  3. Open and render flexboard_01_timeseries.Rmd (provided in today's zip)
Available Components
  • Interactive JavaScript visualizations (htmlwidgets — e.g., plotly, leaflet)
  • Standard R graphical output (base R, ggplot2, lattice)
  • Tabular data with sorting, filtering, paging
  • Value boxes — highlight a single important number
  • Gauges — display a value on a meter within a range
  • Text annotations
Layout by Column (default)

Level-2 markdown headers (##) define columns. Charts stack vertically inside each column.

---
output: flexdashboard::flex_dashboard
---

## Column 1
### Chart A
[code]

## Column 2
### Chart B
[code]

### Chart C
[code]
Layout by Row

Add orientation: rows in the YAML header to stack charts in rows instead of columns.

---
output:
  flexdashboard::flex_dashboard:
    orientation: rows
---

## Row 1
### Chart A

## Row 2
### Chart B
### Chart C
Scrolling Layout

By default, charts resize to fill the browser height. To let charts keep their natural height and scroll the page:

---
output:
  flexdashboard::flex_dashboard:
    vertical_layout: scroll    # scroll instead of fill
---
Section 11

R Shiny — Interactive Web Apps

What is Shiny? Shiny is a web application framework for R. It lets you build interactive web apps using only R — no HTML, JavaScript, or CSS knowledge required (though you can add them). Apps can be published online without a web server.
The 3-Section Structure — Every Shiny App
library(shiny)

# ── SECTION 1: UI ─────────────────────────────────
# Nested R functions that assemble the HTML interface
ui <- fluidPage(
  titlePanel("My App Title"),
  sliderInput("bins",                  # input ID
              "Number of bins:",       # label shown to user
              min = 1, max = 50,
              value = 30),             # default
  plotOutput("distPlot")               # output placeholder
)

# ── SECTION 2: SERVER ─────────────────────────────
# Instructions for building / rebuilding R objects shown in UI
server <- function(input, output) {
  output$distPlot <- renderPlot({
    hist(rnorm(input$bins),            # reacts to slider
         main = "Histogram")
  })
}

# ── SECTION 3: SHINY APP ──────────────────────────
# Combines ui and server — always the last line
shinyApp(ui = ui, server = server)
UI Inputs
  • sliderInput() — numeric slider
  • numericInput() — type a number
  • textInput() — free text
  • selectInput() — dropdown
  • radioButtons() — pick one
  • checkboxGroupInput() — pick many
  • dateInput() — date picker
  • fileInput() — upload a file
  • actionButton() — trigger an action
UI Layout Functions
  • fluidPage() — the outer container
  • sidebarLayout() — sidebar + main panel
  • fluidRow() — row of columns
  • flowLayout() — items flow left to right
  • verticalLayout() — items stacked vertically
  • wellPanel() — a shaded panel box
  • tabsetPanel() — multiple tabs
Server Outputs
  • renderPlot({}) → plotOutput()
  • renderTable({}) → tableOutput()
  • renderPrint({}) → verbatimTextOutput()
  • renderText({}) → textOutput()
Building Your First Shiny App
  1. Install: install.packages("shiny")
  2. In RStudio: File → New File → Shiny Web App → Single File → choose a folder name
  3. R creates a new folder with app.R inside it
  4. Edit app.R — this is the only file you modify
  5. Run it: runApp("path/to/your/app/folder")

In-Class Exercise A — Histograms for Multiple Years (Loop)

Dataset: internet_usage.csv (wide format, years as columns). Create a histogram for each year using a for loop instead of copying and pasting code for every single year.

library(readr)
internet_data <- read_csv("internet_usage.csv")

# Get just the year column names (exclude country text columns)
year_cols <- names(internet_data)[sapply(internet_data, is.numeric)]

# Method 1: for loop
par(mfrow = c(3, 3))   # 3×3 grid of plots (adjust as needed)
for (yr in year_cols) {
  col_data <- internet_data[[yr]]
  n_countries <- sum(!is.na(col_data))   # accurate count
  hist(col_data,
       main  = paste("Internet Usage", yr,
                     paste0("(", n_countries, " countries)")),
       xlab  = "% Population",
       col   = "steelblue",
       border = "white",
       na.rm = TRUE)
}
par(mfrow = c(1, 1))   # reset to single plot
Key pattern to memorize names(df)[sapply(df, is.numeric)] gives you only the column names that are numeric — useful for filtering year columns away from text columns like country names.
In-Class Exercise B — Box Plots Across All Years (While Loop)

Same dataset, same loop logic — but using a while loop and boxplot() instead of hist().

year_cols <- names(internet_data)[sapply(internet_data, is.numeric)]

# Method 2: while loop (alternative to for)
i <- 1
while (i <= length(year_cols)) {
  yr       <- year_cols[i]
  col_data <- internet_data[[yr]]
  n_obs    <- sum(!is.na(col_data))   # real country count

  boxplot(col_data,
          main       = paste("Internet Usage", yr,
                             paste0("(", n_obs, " countries)")),
          ylab       = "% Population Using Internet",
          col        = "#14b8a6",
          border     = "#0d9488",
          horizontal = FALSE)
  i <- i + 1
}
for vs while — when to use which Use a for loop when you know exactly how many times to iterate (you have a fixed vector of years). Use a while loop when the stopping condition is more dynamic (e.g., user input or a changing counter). Both work here — for loops are usually cleaner for this type of task.

Section 01

The Linear Regression Framework

Lakner's Course Philosophy Focus on understanding interpretation and application — computers do the calculations. You need to know what to run, what output to look at, and what it means.
The Players
  • y — Response / Dependent variable (what you predict)
  • x — Predictor / Independent variable (what you use)
  • Simple regression — one predictor
  • Multiple regression — multiple predictors
The Goal

Find the line that best explains y using x — so you can predict future values and understand how much x actually matters.

The Model yᵢ = β₀ + β₁xᵢ + εᵢ
What Each Term Means
TermNameWhat It Is
β₀InterceptPredicted y when x = 0 (sometimes nonsensical)
β₁SlopeChange in y per 1-unit increase in x
εᵢError termRandom noise — what the model can't explain
β̂₀, β̂₁EstimatesWhat R calculates from your data
ŷFitted valueβ̂₀ + β̂₁x — your predicted y
Assumptions About the Error Term ε
  • Normally distributed with mean zero
  • Constant variance (homoscedasticity) — spread doesn't grow with x
  • Independent observations — one error doesn't affect another
Exam interpretation template "For every 1 unit increase in [x], [y] is estimated to increase/decrease by [β̂₁] [units], holding all else constant."
Section 02

Correlation Coefficient (r)

Correlation Formula r = Σ(xᵢ − x̄)(yᵢ − ȳ) / √[Σ(xᵢ − x̄)² · Σ(yᵢ − ȳ)²]
Key property r is always between −1 and +1. It only measures linear relationships. A scatter plot can reveal non-linear patterns that r misses entirely. Always plot first.
Rules of Thumb
r rangeRelationship
0.8 to 1.0Very strong positive
0.5 to 0.8Strong positive
0.2 to 0.5Moderate positive
−0.2 to 0.2No linear rel.
−0.5 to −0.2Moderate negative
−0.8 to −0.5Strong negative
−1.0 to −0.8Very strong negative
Examples from Class
  • Diamonds: r = 0.94 (very strong positive)
  • FCAT Poverty vs Math: r = −0.82 (strong negative)
cor(Diamonds$CARAT, Diamonds$PRICE)
# → 0.94

Red lines = residuals (observed − fitted). Regression minimizes the sum of squared residuals.

Section 03

Least Squares (OLS) Estimation

The Intuition Find the line that makes total prediction error as small as possible. We square errors so positives and negatives don't cancel — and because squares are smooth for calculus to optimize.
Minimize This — SSE SSE = Σ eᵢ² = Σ (yᵢ − ŷᵢ)²
Key Terms
  • ŷᵢ = β̂₀ + β̂₁xᵢ — Fitted (predicted) value
  • eᵢ = yᵢ − ŷᵢ — Residual (leftover error)
  • β̂₀, β̂₁ — R solves for these automatically via OLS
Why Squares Not Absolute Value?

Absolute value has a sharp "kink" at zero — hard to differentiate. Squaring creates a smooth curve that calculus can minimize cleanly. Mathematical convenience, same concept.

Diamond Example — Running the Model
regdiamonds = lm(Diamonds$PRICE ~ Diamonds$CARAT)
regdiamonds

# Or with data frame syntax (cleaner):
model = lm(PRICE ~ CARAT, data=Diamonds)
summary(model)
Estimated Coefficients
β̂₀ (Intercept): −2,298 dollars β̂₁ (Slope): +11,599 dollars per carat
  • Slope interpretation: Each 1-carat increase → price up $11,599. More practical: each 0.1-carat increase → price up ~$1,160.
  • Intercept interpretation: A 0-carat diamond costs −$2,298 — nonsensical. Zero is far outside the data range (min 0.18 carats). Intercept often has no meaningful real-world interpretation.
Extrapolation Warning Never predict y for x values far outside your data range. Run summary(df$x) first to see min/max. The linear relationship is local — it may not hold beyond your data.
Making a Prediction — Plug and Play
ŷ = β̂₀ + β̂₁ × x = −2298 + 11,599 × 0.4 = $2,341.60
summary(Diamonds$CARAT)   # check data range first!
# Min=0.18, Max=1.1 → predicting at 0.4 is fine
Section 04

Reading the R Output — Inference for the Slope

The Key Question Is β₁ = 0? If yes, x has NO linear relationship with y and the model is useless. We test this with a t-test that R gives us automatically.
Hypothesis Test Setup
  • H₀: β₁ = 0 (no relationship)
  • H₁: β₁ ≠ 0 (relationship exists)
  • Reject H₀ if |t_stat| > t₀.₀₂₅
  • Or: reject H₀ if p-value < α (usually 0.05)
  • df = n − 2 for simple regression
The Formulas
t_stat = β̂₁ / s_β̂₁
95% CI: β̂₁ ± t₀.₀₂₅ × s_β̂₁

s_β̂₁ = Std. Error column in summary()

What summary(reg) Gives You — Every Line Explained
Coefficients: Estimate Std. Error t value Pr(>|t|) (Intercept) 189.816 3.021 62.82 <2e-16 *** FCAT$POVERTY -0.305 0.048 -6.42 2.93e-06 *** Residual std error: 5.366 on 20 df Multiple R-squared: 0.6731, Adjusted R-squared: 0.6568 F-statistic: 41.19 on 1 and 20 DF, p-value: 2.927e-06
ColumnWhat to Read
EstimateYour β̂ values — intercept and slope
Std. ErrorUncertainty of the estimate (s_β̂₁) — used to build CI
t valueEstimate ÷ Std. Error — compare to ±t₀.₀₂₅
Pr(>|t|)p-value — if < 0.05, slope is statistically significant
*** / ** / **** = p<0.001 · ** = p<0.01 · * = p<0.05 · "." = p<0.1
Multiple R-squaredR² — proportion of y variation explained by x
Adjusted R-squaredR² penalized for number of predictors — use in multiple regression
Get t Critical Value
qt(0.975, df=20)   # → 2.086

df = n−2. Use 0.975 for 95% two-tailed CI.

Get CI for Slope Directly
confint(reg, level=0.95)

R does the math — gives lower and upper bounds for each coefficient.

Exam shortcut — read the stars *** or ** next to a predictor = significant (reject H₀). No stars = p-value > 0.05 = cannot reject H₀ = that predictor may not belong in the model.
FCAT Poverty/Math — Full Interpretation
  • β̂₀ = 189.8: Average math score for schools with 0% students below poverty is 189.8
  • β̂₁ = −0.305: Each 1% increase in poverty → math score decreases by ~0.3 points
  • t = −6.42, p = 2.93×10⁻⁶: Very strong evidence of a real relationship (p ≪ 0.05)
  • R² = 0.67: 67% of variation in math scores is explained by poverty level
  • Predicting POVERTY=20: ŷ = 189.8 − 0.3 × 20 = 183.8
Section 05

R² and the Decomposition of Variation

The Three Sums of Squares
NameFormulaInterpretation
SST (SS_yy)Σ(yᵢ − ȳ)²Total variation of y around its mean ȳ
SSREGΣ(ŷᵢ − ȳ)²Variation explained by x (regression explains this)
SSEΣ(yᵢ − ŷᵢ)²Variation NOT explained — residuals (random, other factors)
R² = SSREG / SST = 1 − (SSE / SST)
Simple regression only — bonus fact In simple regression, R² = r² (square of correlation coefficient). Diamonds: r = 0.94 → R² = 0.89. In multiple regression this shortcut does NOT apply.
R² Rules of Thumb
R²Model Strength
0.65 to 1.0Strong model — good explaining power
0.25 to 0.65Moderate — OK, room to improve
0 to 0.25Weak — barely worth using in current form
Get SSE/SSREG from R
reg = lm(MATH ~ POVERTY, data=FCAT)
anova(reg)   # shows SSE and SSREG
Adjusted R² — Multiple Regression
  • R² always ↑ when you add predictors (even useless ones)
  • Adjusted R² penalizes for extra predictors
  • Use adjusted R² to compare models with different numbers of predictors
Exam interpretation of R² "R² = 0.67 means 67% of the variation in [math scores] is explained by the linear relationship with [poverty level]." Always name both variables.
Section 06

Confidence Intervals vs Prediction Intervals

Confidence Interval (CI)
  • For the average y at a given x
  • "What is the average price for all 0.6-carat diamonds?"
  • Narrower — gets tighter as n increases
  • Diamond 0.6ct: ($4,535, $4,787)
Prediction Interval (PI)
  • For a single future observation
  • "What will this specific 0.6-carat diamond cost?"
  • Much wider — individual variation always remains
  • Diamond 0.6ct: ($2,458, $6,864)

Blue = regression line · Red bands = confidence · Green bands = prediction (always wider)

R Code — Both Intervals
x = Diamonds$CARAT
y = Diamonds$PRICE
new = data.frame(x = 0.6)

# Confidence interval (average price at CARAT=0.6)
conf = predict(lm(y ~ x), new, interval="confidence")
conf   # → (4534.889, 4787.056)

# Prediction interval (single diamond at CARAT=0.6)
pred = predict(lm(y ~ x), new, interval="prediction")
pred   # → (2458, 6864)
Common mistake Must save x and y first, then use data.frame(x = 0.6). Writing data.frame(Diamonds$CARAT = 0.6) does NOT work.
Plot Both Bands
pred = predict(model, interval="prediction")
confid = predict(model, interval="confidence")

plot(x, y)
lines(x, pred[,1], col="blue", lwd=2)    # regression line
lines(x, pred[,2], col="green", lwd=2)   # PI lower bound
lines(x, pred[,3], col="green", lwd=2)   # PI upper bound
lines(x, confid[,2], col="red", lwd=2)   # CI lower bound
lines(x, confid[,3], col="red", lwd=2)   # CI upper bound
Which interval on the exam? "Average" or "expected" value → confidence interval (CI). "A single specific" observation → prediction interval (PI). PI is always wider than CI for the same x.
Section 07

Multiple Regression & Multicollinearity

Multiple Regression Model y = β₀ + β₁x₁ + β₂x₂ + … + βₖxₖ + ε
Coefficient interpretation in multiple regression Each β̂ᵢ = "holding all other predictors constant, a 1-unit increase in xᵢ is associated with a β̂ᵢ change in y."
Zagat Restaurant Example
datazagat = read.table("zagat.CSV", header=TRUE, sep=",")
zagatmod = lm(Price ~ Food + Decor + Service, data=datazagat)
summary(zagatmod)
Estimate Std. Error t value Pr(>|t|) (Intercept) -21.195 2.271 -9.334 <2e-16 *** Food -0.107 0.166 -0.648 0.518 Decor 1.008 0.127 7.955 <2e-16 *** Service 2.603 0.233 11.197 <2e-16 *** Multiple R-squared: 0.7765
  • Food: negative and non-significant (p=0.518) — strange! Counterintuitive but it's a statistical artifact, not reality.
  • Decor and Service: both *** significant
  • Food p-value > 0.05 → cannot reject H₀ for Food → consider removing it
Multicollinearity — The Real Problem
data = datazagat[, c(2,3,4,5)]
cor(data)
Food Decor Service Decor 0.363 Service 0.710 0.726 Price 0.538 0.779 0.849
  • Food–Service correlation = 0.71 — Food is redundant because Service already "captures" it
  • When predictors correlate with each other, their coefficients become unstable and hard to interpret
  • This doesn't mean Food doesn't affect Price — it just can't add unique info given Service is in the model
  • Rule: Predictors should have LOW correlation with each other, HIGH correlation with y
Best Subset Selection — LEAPS Package
library(leaps)
leaps = regsubsets(Price ~ Food + Decor + Service, data=datazagat, nbest=3)
plot(leaps, scale="r2")
Reading the LEAPS plot Black squares = predictor included. Find row with highest R² using fewest predictors. In Zagat: Decor + Service only gives R²≈0.78, nearly identical to the full model (0.7765). Drop Food.
Voting Fraud Case Study — A Statistical Warning
Statistical Misuse — Know This Concept You cannot select data because it looks unusual, then use statistics to "prove" it's unusual. That's circular reasoning. Random sampling is required for valid probability statements.
  • Context: 1993 Philadelphia election — absentee ballot ratio = 3.76 (suspicious), machine ratio = 0.97
  • Method: Fit regression on 21 prior elections, built prediction interval for ratioabs given ratiomach = 0.97
  • Result: 3.76 fell outside even the 99.9% PI → very suspicious
  • The flaw: This election was selected because it looked unusual — not a random selection
  • Better approach: Probability of seeing 1 such unusual event in 22 tries = 1 − (1−0.001)²² = 0.022. Still very small. Fraud conclusion still reasonable but reasoning must be sound.
Multicollinearity exam rule Predictors should explain y but NOT each other. Use cor(data) to check. If two predictors are highly correlated (>0.7), consider dropping the one with the higher p-value.
