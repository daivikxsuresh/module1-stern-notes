Section 03

Data Project Fundamentals

6 Questions to Ask About Any New Dataset
  1. How many observations (rows) and variables (columns) are there?
  2. What are the data types of each column (numeric, character, factor)?
  3. Are there missing values — and if so, what do they mean?
  4. What does each row represent? (the entity / unit of analysis)
  5. Where did the data come from and can you trust it?
  6. What time period does it cover?
The Process
  • Data represents real-world phenomena — always ask "what does a row mean?"
  • 80% of the work is in preparation and tidying
  • Only 20% is analysis and visualization
Quick Exploration Code
nrow(df)     # number of rows
ncol(df)     # number of cols
dim(df)      # rows × cols
str(df)      # types + preview
head(df)     # first 6 rows
summary(df)  # stats per column
Section 04

dplyr — Data Manipulation Package

What is dplyr? dplyr is R's equivalent of Python's pandas. It gives you a clean, readable grammar for manipulating data frames. It's part of the tidyverse. Always load it with library(dplyr).
FunctionWhat it doesExample
select()Choose columns to keepselect(df, col1, col2)
filter()Keep rows that match a conditionfilter(df, amount > 100)
mutate()Create a new column (or overwrite one)mutate(df, ratio = a/b)
summarize()Collapse rows into summary statisticssummarize(df, avg = mean(x))
group_by()Group rows before summarizinggroup_by(df, region)
arrange()Sort rowsarrange(df, desc(amount))
The Pipe Operator |>

The pipe passes the left-hand result as the first argument to the right-hand function. It makes multi-step operations read like a sentence.

library(dplyr)

# Without pipe (hard to read)
arrange(summarize(group_by(filter(df, region == "East"), region),
  total = sum(sales)), desc(total))

# With pipe (clean and readable)
df |>
  filter(region == "East") |>
  group_by(region) |>
  summarize(total = sum(sales)) |>
  arrange(desc(total))
select() — Pick Columns
library(readr)
sales <- read_csv("salesdata.csv")

# Keep only two columns
sales |> select(region, amount)

# Drop a column with minus
sales |> select(-id)
filter() — Subset Rows
# Single condition
sales |> filter(amount > 500)

# Multiple conditions (AND)
sales |> filter(amount > 500,
                region == "West")

# OR condition
sales |> filter(region == "East" |
                region == "West")
mutate() — New Columns
# Create a new column
sales |> mutate(tax = amount * 0.08)

# Multiple new columns at once
sales |> mutate(
  tax    = amount * 0.08,
  total  = amount + tax
)
group_by() + summarize()
# Total sales by region
sales |>
  group_by(region) |>
  summarize(
    total = sum(amount),
    avg   = mean(amount),
    n     = n()
  )

Tidying Wide → Long (Pivot)

The internet usage data comes in wide format — each year is its own column. We need to pivot it to long (tidy) format so ggplot2 can work with it.

library(readr)
library(dplyr)
library(tidyr)

internet_readr <- read_csv("internet_usage.csv", skip = 0)

# After selecting only countries (removing aggregates),
# pivot from wide to long
tidy_internet_readr <- internet_readr |>
  pivot_longer(
    cols      = -c(country, iso2c, iso3c),
    names_to  = "year",
    values_to = "usage"
  ) |>
  mutate(year = as.numeric(year))

# Result: 4 columns — country, iso3c, year, usage

ggplot2 — Data Visualization

ggplot2 grammar Every ggplot is built the same way: ggplot(data, aes(x, y, color, group)) + geom_*() + .... The aes() function maps columns to visual properties. You add layers with +.
Line Chart — Internet Usage Over Time
library(ggplot2)
library(ggthemes)

line01 <- ggplot(tidy_internet_readr,
                 aes(x     = year,
                     y     = usage,
                     color = country,
                     group = country)) +
  geom_line() +
  labs(title = "World Internet Usage",
       x     = "Year",
       y     = "% of Population") +
  theme_economist()

line01   # render the plot
Bar Chart — Bar / Column
bar01 <- ggplot(tidy_internet_readr,
                aes(x = country, y = usage)) +
  geom_col() +
  labs(title = "Internet Usage by Country") +
  theme_minimal()

bar01
geom_bar vs geom_col Use geom_bar() when you want R to count rows for you. Use geom_col() when your data already has the height values as a column. For this dataset, use geom_col().
Making it Interactive with plotly
library(plotly)

# Wrap any ggplot object with ggplotly()
ggplotly(line01)    # hover, zoom, pan — all automatic

ggplotly converts any ggplot into an interactive HTML widget with no extra code.

Section 07

Writing Custom Functions

Why write your own functions? When you need to repeat the same operation many times, wrap it in a function instead of copying and pasting code. Functions also make your code easier to test and reuse.
Function Structure
# Part 1 — define the function
myfunction <- function(myparameter) {
  return(myparameter + 2)
}

# Part 2 — call the function
myfunction(5)    # → [1] 7
myfunction(10)   # → [1] 12
Adding Input Validation

What happens if someone passes a string instead of a number? Add a check with is.numeric():

myfunction <- function(myparameter) {
  if (!is.numeric(myparameter)) {
    stop("Input must be numeric")
  }
  return(myparameter + 2)
}

myfunction("hello")   # → Error: Input must be numeric
myfunction(8)         # → [1] 10
Multiple Arguments
# Function with two parameters
addTogether <- function(x, y) {
  return(x + y)
}

addTogether(3, 4)          # → [1] 7
addTogether(x=3, y=4)      # same — named arguments (alternative)
addTogether(3, "hello")    # → Error (type mismatch)
Built-in Functions You Should Know
FunctionWhat it does
toupper(x)Convert string to uppercase
mean(x, na.rm=TRUE)Average, skip NAs
is.numeric(x)Returns TRUE/FALSE — is x a number?
is.na(x)Returns TRUE/FALSE — is x missing?
sqrt(x)Square root
Pseudocode → Real Code Workflow

Before writing code, sketch the logic in plain English (pseudocode), then translate:

# Pseudocode:
# function "averageTwo" takes two numbers x and y
# adds them together, divides by 2, returns result

averageTwo <- function(x, y) {
  if (!is.numeric(x) | !is.numeric(y)) stop("Both inputs must be numeric")
  return((x + y) / 2)
}

averageTwo(10, 20)   # → [1] 15
Section 08

sapply — Apply a Function Over a Vector

The apply family Instead of writing a for loop to apply a function to every element, R has the apply family. sapply() applies a function over a vector or list and returns a simplified result (usually a vector).
sapply() Syntax
# Pattern: sapply(vector_or_list, function)

# Define a function
takemean <- function(x) { mean(x, na.rm = TRUE) }

# Apply it over a named list of vectors
my_data <- list(a = c(1,2,3), b = c(10,20), c = c(5,5,5,5))
sapply(my_data, takemean)
## a b c ## 2 15 5

Section 09

Flexdashboards

What is a Flexdashboard? A Flexdashboard is an R Markdown document that publishes a group of related visualizations as a dashboard web page. It can be static (plain HTML) or dynamic (powered by Shiny).
Setup
  1. Install the package: install.packages("flexdashboard")
  2. In RStudio: File → New File → R Markdown → From Template → Flex Dashboard
  3. Open and render flexboard_01_timeseries.Rmd (provided in today's zip)
Available Components
  • Interactive JavaScript visualizations (htmlwidgets — e.g., plotly, leaflet)
  • Standard R graphical output (base R, ggplot2, lattice)
  • Tabular data with sorting, filtering, paging
  • Value boxes — highlight a single important number
  • Gauges — display a value on a meter within a range
  • Text annotations
Layout by Column (default)

Level-2 markdown headers (##) define columns. Charts stack vertically inside each column.

---
output: flexdashboard::flex_dashboard
---

## Column 1
### Chart A
[code]

## Column 2
### Chart B
[code]

### Chart C
[code]
Layout by Row

Add orientation: rows in the YAML header to stack charts in rows instead of columns.

---
output:
  flexdashboard::flex_dashboard:
    orientation: rows
---

## Row 1
### Chart A

## Row 2
### Chart B
### Chart C
Scrolling Layout

By default, charts resize to fill the browser height. To let charts keep their natural height and scroll the page:

---
output:
  flexdashboard::flex_dashboard:
    vertical_layout: scroll    # scroll instead of fill
---
Section 10

R Shiny — Interactive Web Apps

What is Shiny? Shiny is a web application framework for R. It lets you build interactive web apps using only R — no HTML, JavaScript, or CSS knowledge required (though you can add them). Apps can be published online without a web server.
The 3-Section Structure — Every Shiny App
library(shiny)

# ── SECTION 1: UI ─────────────────────────────────
# Nested R functions that assemble the HTML interface
ui <- fluidPage(
  titlePanel("My App Title"),
  sliderInput("bins",                  # input ID
              "Number of bins:",       # label shown to user
              min = 1, max = 50,
              value = 30),             # default
  plotOutput("distPlot")               # output placeholder
)

# ── SECTION 2: SERVER ─────────────────────────────
# Instructions for building / rebuilding R objects shown in UI
server <- function(input, output) {
  output$distPlot <- renderPlot({
    hist(rnorm(input$bins),            # reacts to slider
         main = "Histogram")
  })
}

# ── SECTION 3: SHINY APP ──────────────────────────
# Combines ui and server — always the last line
shinyApp(ui = ui, server = server)
UI Inputs
  • sliderInput() — numeric slider
  • numericInput() — type a number
  • textInput() — free text
  • selectInput() — dropdown
  • radioButtons() — pick one
  • checkboxGroupInput() — pick many
  • dateInput() — date picker
  • fileInput() — upload a file
  • actionButton() — trigger an action
UI Layout Functions
  • fluidPage() — the outer container
  • sidebarLayout() — sidebar + main panel
  • fluidRow() — row of columns
  • flowLayout() — items flow left to right
  • verticalLayout() — items stacked vertically
  • wellPanel() — a shaded panel box
  • tabsetPanel() — multiple tabs
Server Outputs
  • renderPlot({}) → plotOutput()
  • renderTable({}) → tableOutput()
  • renderPrint({}) → verbatimTextOutput()
  • renderText({}) → textOutput()
Building Your First Shiny App
  1. Install: install.packages("shiny")
  2. In RStudio: File → New File → Shiny Web App → Single File → choose a folder name
  3. R creates a new folder with app.R inside it
  4. Edit app.R — this is the only file you modify
  5. Run it: runApp("path/to/your/app/folder")

library(readr)
library(dplyr)
library(stringr)

nyuclasses <- read_csv("nyuclasses/nyuclasses.csv") |>
  filter(str_detect(assessment, "^Assignment"))

ui <- fluidPage(
  titlePanel("NYU Classes Dashboard"),

  sidebarLayout(
    sidebarPanel(
      # ── Add your inputs here ──────────────────
      selectInput("assignment",
                  "Select Assignment:",
                  choices = unique(nyuclasses$assessment)),

      # Example: student selector (Exercise 04 enhancement)
      selectInput("student",
                  "Filter by Student:",
                  choices = c("All", unique(nyuclasses$student)),
                  multiple = TRUE,
                  selected = "All")
    ),

    mainPanel(
      plotOutput("distPlot")
      # Add more outputs here for doughnut charts, tables, etc.
    )
  )

server <- function(input, output) {
  # Reactive data filtered by inputs
  filtered <- reactive({
    df <- nyuclasses |>
      filter(assessment == input$assignment)
    if (!"All" %in% input$student) {
      df <- df |> filter(student %in% input$student)
    }
    df
  })

  output$distPlot <- renderPlot({
    ggplot(filtered(), aes(x = score)) +
      geom_histogram(fill = "#0d9488", color = "white", bins = 20) +
      labs(title = paste("Distribution:", input$assignment)) +
      theme_minimal()
  })
}

shinyApp(ui = ui, server = server)
